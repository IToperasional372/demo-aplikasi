Sistem Laporan Inventori Gudang Realtime Terintegrasi Firebase & Google Sheets

1. Tentang LogiStore Executive

LogiStore Executive adalah sistem informasi manajemen inventori gudang realtime eksekutif yang dirancang khusus untuk memangkas birokrasi operasional dan meniadakan biaya server bulanan. Dengan menggabungkan teknologi modern Firebase Cloud Database (Firestore) di frontend untuk akses kilat, serta Google Sheets di backend untuk kemudahan pengolahan data, aplikasi ini menawarkan sinkronisasi mandiri tanpa biaya berlangganan server (*Zero Monthly Hosting Cost*).

2. Alur Kerja Data & Sinkronisasi

Aplikasi ini beroperasi dalam ekosistem sinkronisasi tiga arah yang otomatis dan aman:

🖥️ Web App Inventori (Peramban / Browser Gudang)

➔ Mengirim, memproses, dan membarui mutasi data langsung ke Firebase Firestore secara realtime dalam hitungan milidetik.

💾 Google Sheets Database (Admin / Kantor Pusat)

➔ Menarik seluruh transaksi atau data mutasi dari Firebase secara live menggunakan integrasi Apps Script bawaan Google Sheets.

3. Memahami Dasbor Ringkasan

Dasbor Ringkasan menyajikan ringkasan visual tingkat tinggi (*executive metrics*) yang akurat untuk pemantauan cepat:

  • Total SKU Produk: Jumlah keseluruhan variasi atau tipe barang unik yang terdaftar aktif di dalam database gudang.
  • Stok Aman: Kuantitas barang yang persediaan fisiknya saat ini masih berada di atas batas aman minimal yang telah ditentukan.
  • Stok Kritis: Jumlah barang yang persediaannya telah menyentuh batas aman atau habis (0 unit), yang wajib segera dilakukan pemesanan ulang (*restock*).
  • Estimasi Nilai Aset: Kalkulasi nilai modal bersih persediaan di gudang secara otomatis (jumlah stok fisik dikalikan harga beli satuan produk) secara realtime.

4. Manajemen Stok & Inventori

Menu **Stok & Inventori** merupakan modul utama untuk administrasi fisik barang. Anda dapat mengoperasikan beberapa fungsi utama:

  1. Pendaftaran Barang Baru: Klik tombol merah *"+ Barang Baru"*, lengkapi identitas (SKU, nama barang, kategori, stok awal, batas minimal kritis, lokasi rak, dan harga unit), lalu simpan.
  2. Penyesuaian Mutasi Stok (Paling Penting): Klik tombol **"Mutasi"** pada barang yang diinginkan. Tentukan arah pergerakan barang (Stok Masuk / Stok Keluar), input jumlah unit beserta alasan referensi, lalu konfirmasi. Sistem akan mengupdate stok fisik dan mencatat log mutasi secara otomatis demi kepatuhan audit.
  3. Ubah / Hapus Data: Klik tombol ikon pensil untuk memperbarui deskripsi barang tanpa merusak data histori transaksi, atau tombol ikon tempat sampah untuk menghapus barang.

5. Penggunaan Fitur Scan Barcode Kamera Live

LogiStore meniadakan pengetikan SKU manual yang lambat dan rawan *human-error*. Ikuti panduan pemindaian berikut:

  1. Masuk ke tab menu **Stok & Inventori**.
  2. Klik tombol berwarna oranye **"Scan Barcode"** di kanan atas.
  3. Izinkan peramban (browser) Anda untuk mengakses kamera jika muncul pesan perizinan keamanan (*permission pop-up*).
  4. Arahkan kamera pada kode batang (barcode) atau QR Code produk fisik Anda.
  5. Logika Deteksi Cerdas:
    • Jika SKU dikenali oleh sistem, modal **Mutasi Stok** untuk barang tersebut akan langsung terbuka secara otomatis.
    • Jika SKU belum terdaftar, sistem akan memberikan tawaran instan untuk mendaftarkan barang baru dengan kode SKU yang sudah terisi otomatis sesuai hasil scan.

6. Ekspor Laporan Excel & Cetak PDF Resmi

Membuat pelaporan fisik siap audit menjadi sangat mudah dengan fitur ekspor instan:

  • Ekspor Excel (CSV): Klik tombol **"📥 Excel (CSV)"** untuk mengunduh seluruh persediaan aktif dalam format file `.csv`. File ini menggunakan pembatas (*delimiter*) standar yang langsung terbaca rapi oleh Microsoft Excel atau Google Sheets.
  • Cetak Laporan PDF Resmi: Klik tombol **"🖨️ Cetak PDF"**. Melalui CSS `@media print`, website akan otomatis menyembunyikan elemen dekoratif web (seperti navigasi, tombol edit, dsb.) dan memformat tabel data menjadi dokumen cetak hitam-putih formal yang siap dicetak ke kertas fisik atau disimpan sebagai berkas PDF resmi.

7. Setup Developer: Firebase Spark Plan

Ikuti langkah berikut untuk mengonfigurasi basis data Firebase (Firestore & Authentication) secara gratis:

  1. Buka [Firebase Console](https://console.firebase.google.com/) dan buat proyek baru dengan nama proyek pilihan Anda.
  2. Aktifkan **Anonymous Sign-in** pada menu *Authentication* > *Sign-in method* agar pengguna terautentikasi secara aman di latar belakang.
  3. Buat database baru di menu **Firestore Database** menggunakan server lokasi terdekat (misalnya `asia-southeast2` di Jakarta).
  4. Masuk ke tab **Rules** pada Firestore Database, dan ubah kodenya menjadi seperti berikut agar akses data terproteksi: 
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /artifacts/{appId}/public/data/{collectionName}/{document} {
      allow read, write: if request.auth != null;
    }
  }
}

8. Sinkronisasi Google Sheets

Menghubungkan data Firebase agar dapat ditarik (*import*) atau dikirim (*export*) langsung dari spreadsheet Google Sheets Anda:

  1. Buka file Google Sheets kosong Anda.
  2. Pilih menu **Ekstensi** > **Apps Script**.
  3. Salin seluruh kode dari file **`apps_script_sync.js`** yang telah kami sediakan, lalu tempelkan ke dalam editor Apps Script.
  4. Ubah nilai `API_KEY` dan `PROJECT_ID` di bagian atas script sesuai dengan proyek Firebase Anda.
  5. Simpan, lalu segarkan halaman Google Sheets Anda. Menu baru bernama **"🔥 Firebase Gudang"** akan muncul otomatis di samping menu bantuan, siap digunakan untuk melakukan sinkronisasi data dua arah!
